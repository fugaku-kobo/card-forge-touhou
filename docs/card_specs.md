# カード仕様(東方版・v3.1 / 2リーダー制)

カードのデータ構造定義。**Excel インポート / カード制作フォーム / プレイ判定** の共通参照点。

最終更新: 2026-05-22
状態: **v3.1 対応**(`docs/rules.md` v3.1・2リーダー制と同期)

---

# 1. 共通フィールド

すべてのカードに共通:

| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | string | ◯ | カード一意 ID(エディタ内部発行) |
| `name` | string | ◯ | カード名 |
| `type` | enum | ◯ | `'キャラ'` / `'スペル'`(2 種) |
| `work` | string | × | 作品名・出典(例: 「東方紅魔郷」) |
| `cardCode` | string | × | カード型番(例: `TH-001`) |
| `icon` | string | × | 絵文字アイコン(手動指定、空欄なら自動推測) |
| `art` | string | × | イラストメモ(絵の内容) |
| `unlimited` | bool | × | true なら同 ID 上限(5 枚)を無視 |

> v3 で「シューティング」タイプは廃止。カードタイプは **キャラ / スペルの 2 種**。

# 2. タイプ別フィールド

## 2.1. キャラカード(= リーダー。`type = 'キャラ'`、`kind = 'touhou-char'`)

試合前に 2 体選び、リーダーゾーンに固定する本体カード。
> 制作ツール実装上は、プレイ用デッキにキャラを 2 枚入れておくと、ゲーム開始時に場(リーダーゾーン)へ自動抽出される。

| キー | 型 | 必須 | xlsx 対応列 | 説明 |
|---|---|---|---|---|
| `level` | int 1-3 | × | レベル | 通常時 Lv。スペルの発動条件 Lv の参照用(既定 1) |
| `attack` | int(文字列) | ◯ | 攻撃力 | 通常時攻撃力 |
| `evasion` | int(文字列) | ◯ | 回避力 | 通常時回避力 |
| `effect` | string | × | (なし) | 通常時の効果文(常時・攻撃時・迎撃時など) |
| `awakenCost` | int(文字列) | × | 覚醒コスト | 覚醒(裏返し)に必要な P |
| `awakenLevel` | int 1-3 | × | 覚醒時Lv | 覚醒後の Lv(通常時 Lv 以上。既定 2) |
| `awakenAttack` | int(文字列) | × | 覚醒時攻撃力 | 覚醒後の攻撃力 |
| `awakenEvasion` | int(文字列) | × | 覚醒時回避力 | 覚醒後の回避力 |
| `awakenEffect` | string | × | 覚醒時能力 | 覚醒した瞬間に発動する効果 |

## 2.2. スペルカード(`type = 'スペル'`、`kind = 'touhou-spell'`)

デッキ 80 枚を構成する唯一のカードタイプ。

| キー | 型 | 必須 | xlsx 対応列 | 説明 |
|---|---|---|---|---|
| `spellKind` | string | × | 種類 | `スペルカード`(通常)/ `QUICK`(バトル中の割り込み可) |
| `requiredCharLevel` | int 1-3 / '' | × | 発動条件Lv | 自分の **どちらかのリーダー** の現在 Lv がこの値以上で使用可 |
| `ppCost` | int(文字列) | × | PPコスト | P ゾーンから消費する量 |
| `bomCost` | int(文字列) | × | BOMコスト | BOMB ゾーンから消費する量 |
| `trigger` | string | × | トリガー | 発動できるタイミング(自由テキスト。例: 「バトル中」) |
| `recoverP` | int(文字列) | × | 回収P | 攻撃でトラッシュ送りされた時、攻撃側が獲得(1 攻撃あたり上限 3) |
| `effect` | string | ◯ | 効果 | カード効果文 |

> 旧 v2 の `user`(使用者)・`costKind` 等は廃止。スペルは特定リーダーに紐づかない。

# 3. プレイ時の動的状態(instance レベル)

カード定義ではなく、PLAY タブで個別カードに付くフィールド:

| キー | 型 | 説明 |
|---|---|---|
| `instanceId` | string | プレイ中の個別管理 ID |
| `cardId` | string | 参照するカード定義の id |
| `zone` | enum | 現在のゾーン(`deck`/`hand`/`field`/`trash`/`p`/`life`/`bomb`) |
| `faceDown` | bool | 裏向きか(ライフ・BOMB は裏向き。リーダーは表向き) |
| `awakened` | bool | リーダーが覚醒状態か(true なら覚醒時ステータス・能力を参照) |
| `rested` | bool | リーダーが行動済みか(攻撃後 true、次の自分のターン開始で false) |

> `field` ゾーンはリーダー 2 体を保持する。

# 4. Excel インポートフォーマット

## シート構成
- シート名で type 判定: `キャラカード` → キャラ / `スペルカード` → スペル
- 旧「シューティング」シートは **スペル** として取り込む(v3 で型統合)

## キャラシート列
`名前 | (空) | 攻撃力 | 回避力 | 覚醒コスト | 覚醒時攻撃力 | 覚醒時回避力 | 覚醒時能力`

## スペルシート列
`カード名 | 種類 | PPコスト | BOMコスト | トリガー | 効果`

実装側追加列(将来):`ID | 発動条件Lv | 回収P | レベル / 覚醒時Lv | 作品名 | 絵文字 | 無制限フラグ`
