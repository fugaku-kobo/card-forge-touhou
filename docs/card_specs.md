# カード仕様(東方版・覚醒モデル)

カードのデータ構造定義。**Excel インポート / カード制作フォーム / プレイ判定** の共通参照点。

最終更新: 2026-05-19
状態: **ドラフト**(覚醒モデル v2 対応、未確定フィールドあり)

---

# 1. 共通フィールド

すべてのカードに共通:

| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | string | ◯ | カード一意 ID(エディタ内部発行) |
| `name` | string | ◯ | カード名 |
| `type` | enum | ◯ | `'キャラ'` / `'スペル'` / `'シューティング'` |
| `work` | string | × | 作品名・出典(例: 「東方紅魔郷」) |
| `cardCode` | string | × | カード型番(例: `TH-001`) |
| `icon` | string | × | 絵文字アイコン(手動指定、空欄なら自動推測) |
| `art` | string | × | イラストメモ(絵の内容) |
| `unlimited` | bool | × | true なら同 ID 上限を無視 |

# 2. タイプ別フィールド

## 2.1. キャラカード(`type = 'キャラ'`、`kind = 'touhou-char'`)

| キー | 型 | 必須 | xlsx 対応列 | 説明 |
|---|---|---|---|---|
| `level` | int 1-3 / '' | × | (なし) | 任意。スペル発動条件 Lv の参照用 |
| `attack` | int(文字列) | ◯ | 攻撃力 | 通常時攻撃力 |
| `evasion` | int(文字列) | ◯ | 回避力 | 通常時回避力 |
| `effect` | string | × | (なし) | 通常時の効果文(登場時・常時など) |
| `awakenCost` | int(文字列) | × | 覚醒コスト | 覚醒(裏返し)に必要な P |
| `awakenAttack` | int(文字列) | × | 覚醒時攻撃力 | 覚醒後の攻撃力 |
| `awakenEvasion` | int(文字列) | × | 覚醒時回避力 | 覚醒後の回避力 |
| `awakenEffect` | string | × | 覚醒時能力 | 覚醒時に発動する効果(例: 「1ドロー」) |

旧 v1 のフィールド `evolveFrom` / `evolveCost` は **廃止**。

## 2.2. スペルカード(`type = 'スペル'`、`kind = 'touhou-spell'`)
## 2.3. シューティングカード(`type = 'シューティング'`、`kind = 'touhou-shoot'`)

スペルとシューティングは **同一フィールド構成** を共有(xlsx 同等):

| キー | 型 | 必須 | xlsx 対応列 | 説明 |
|---|---|---|---|---|
| `user` | string | × | 使用者 | スペルが紐づくキャラ名(例: 「霊夢」)。プレイ判定に使うかは **未定** |
| `spellKind` | string | × | 種類 | `スペルカード` / `シューティング` / `QUICK` 等(自由テキスト) |
| `requiredCharLevel` | int 1-3 / '' | × | (PDF: 左上) | 発動条件 Lv |
| `ppCost` | int(文字列) | × | PPコスト | P ゾーンから消費する量 |
| `bomCost` | int(文字列) | × | BOMコスト | BOMB ゾーンから消費する量(PP と両建て可) |
| `trigger` | int(文字列) | × | トリガー | xlsx では数値(1, 2)。意味は **未定**(発動条件Lv の別表記?能力レベル?) |
| `recoverP` | int(文字列) | × | (PDF: 右下青) | 攻撃でトラッシュ送りされた時、攻撃側が獲得 |
| `effect` | string | ◯ | 効果 | カード効果文 |

旧 v1 のフィールド `costKind` / `costAmount` は **廃止**。

# 3. 未確定フィールド(`docs/rules.md` §J と同期)

`rules.md` の未確定事項のうち、データ構造に影響するもの:

| 項目 | 影響 |
|---|---|
| キャラの場出しコスト(`playCost`?) | xlsx に列なし。初期 1 枚配置のみなら不要 |
| スペル種別の網羅 | enum 化するか、自由テキストで保持するか |
| Lv の範囲 | 1〜3 か、なしも許容するか |
| `user` の用途 | プレイ判定に使うか(使用者キャラが盤面にいないと発動不可 等)、レコードのみか |
| `trigger` の意味 | 数値ベースだが意味未確定(発動条件Lv との関係) |
| スペル と シューティング の差異 | フィールド構成は同じ。プレイ上の挙動が変わるか **未定** |

# 4. プレイ時の動的状態(`instance` レベル)

カード定義そのものではなく、PLAY タブで個別カードに付くフィールド(手塚版踏襲・東方版で再整理):

| キー | 型 | 説明 |
|---|---|---|
| `instanceId` | string | プレイ中の個別管理 ID |
| `cardId` | string | 参照するカード定義の id |
| `zone` | enum | 現在のゾーン(`'deck'` / `'hand'` / `'field'` / `'trash'` / `'p'` / `'life'` / `'bomb'`) |
| `faceDown` | bool | 裏向きか(セットアップ時の盤面・ライフ・BOMB が裏向き) |
| `awakened` | bool | キャラが覚醒状態か(true なら覚醒時ステータス・能力を参照) |

> 手塚版の `inst.hp`(現 HP)・進化スタックは東方版では **不要**。覚醒は単一フラグで表現。

# 5. Excel インポートフォーマット(計画)

スターターデッキ 2 種の Excel テンプレートで以下を想定:

## 共通シート構成
- 1 シート = 1 カードタイプ
- シート名で type 判定:
  - `キャラカード` → `キャラ`
  - `スペルカード` → `スペル`
  - `シューディングカード` / `シューティングカード` → `シューティング`

## キャラシート想定列(xlsx 準拠)
`名前 | (空) | 攻撃力 | 回避力 | 覚醒コスト | 覚醒時攻撃力 | 覚醒時回避力 | 覚醒時能力`

実装側追加列(将来):`ID | 効果(通常時) | レベル | 作品名 | 絵文字 | 無制限フラグ`

## スペル / シューティング シート想定列(xlsx 準拠)
`カード名 | 使用者 | 種類 | PPコスト | BOMコスト | トリガー | 効果`

実装側追加列(将来):`ID | 発動条件Lv | 回収P | 作品名 | 絵文字 | 無制限フラグ`

詳細は Phase 5(Excel インポート実装時)に詰める。
